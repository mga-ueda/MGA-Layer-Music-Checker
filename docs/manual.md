# MGA Layer Music Checker

多レイヤー音声（最大 4 トラック）の同期再生・ミックス・スペクトラム／メーター表示をブラウザ内だけで行う Web アプリです。音声ファイルはサーバーへ送信されません。

## クイックスタート

1. リポジトリを取得し、`index.html` と `css/`・`js/` を同じフォルダ構成のまま置く
2. **Google Chrome** で `index.html` を開く（ローカル HTTP サーバー経由を推奨）
3. 破線のドロップゾーンに WAV / MP3 / AIF（AIFF）をドロップして利用する

開発時は Cursor / VS Code で **F5**（「ブラウザでデバッグ」）を使うと、`py -3 -m http.server` でローカルサーバーが起動します。詳細は `.vscode/launch.json` を参照してください。

## リポジトリ構成

| パス | 内容 |
|------|------|
| `index.html` | アプリ本体（UI） |

取扱説明を直すときは **`docs/manual.md` のみ編集**し、次を実行してください。

```bash
py -3 scripts/sync-manual.py
```

## 使い方

### ブラウザとプライバシー

- Google Chrome の利用を推奨します。ブラウザによっては形式ごとにデコードできないことがあります。
- 音声はこのブラウザ内でのみデコード・再生され、いかなる情報もサーバーへ送信されることはありません。

### ファイルの読み込み

- 破線のドロップゾーンに WAV / MP3 / AIF（AIFF）をドラッグ＆ドロップします。複数トラックを同時に選んでドロップすると、最大 4 トラックまで、ファイル名ソート順でスロット 1〜4 に読み込まれます。1 ファイルだけドロップした場合はスロット 1 のみです。複数スロットを使うには、OS のファイル一覧で複数選択してから一度の操作でドロップしてください。
- ディスク上の元ファイルを編集したり書き出しし直した場合でも、本アプリは自動では再読み込みしません。更新した音声を反映するには、必ずドロップゾーンへドロップし直してください。次回起動後に PLAY だけで戻るのは、前回保存されたデータのコピーです（保存の仕組みは特記事項を参照）。

### 再生・ミキシング・表示

- 読み込み成功後、スロット 1 のフェーダーは MAX に設定され、同期再生が自動で開始します。スロット 2〜4 はフェーダーを上げるまで最小のままです。
- PLAY SYNC ボタンで再生の開始／停止。再生中は全トラックが同期され、シークバーと時間の最大値は、読み込んだトラック内で最短の長さのものが利用されます。
- 縦型のフェーダーで各チャンネル音量を設定します。Fade IN / OUT は、トランスポート行の フェード時間（1〜30 秒の整数のみ）をかけてフェーダーを変化させます。フェード中に同じスロットへ再度フェードをかけた場合は、直前のフェードに使っていた秒数を切り上げて半分にした長さ（最短 1 秒）で新しいフェードを行います（フェードが加速するような挙動になります）。この操作によるデフォルト値の上書きはされません。
- 画面上の Spectrum floor / Level meter floor / トランスポート付近の フェード時間（秒）で、スペクトラム／メーターの表示下限やフェード長を変更できます。
- クリップが検出されると、自動的にマスターボリュームが減衰し、クリップ表示が点灯します。マスターボリューム値は、次回起動時には 100% にリセットされます。

### キーボードショートカット

- キーボードショートカットが利用できます。テキスト／数値入力にフォーカスがある間はキーボードショートカットは無効です。アプリ画面上のキーボードショートカット表を参照してください。

### データの保存とパフォーマンス

- 各スロットの音量・Mute・Solo などのミックス状態と、読み込んだ音声のコピーは、ドロップ直後およびミックス操作のあと IndexedDB に保存され、次回以降 PLAY で前回トラックを復元するときに使われます（HTTP のディスクキャッシュではなく、このアプリ用の保存領域です）。既定の初期値や選択肢のリストをソース内で変更することもできます。変更手順は、「ソースコードを改変する場合」を参照してください。
- スペクトラム表示下限・レベルメーター表示下限・フェード時間（秒）を変更すると、ブラウザのローカルストレージに保存され、次回このページを開いたときに復元されます。
- ドロップで読み込んだ音声は、その時点のファイル内容がブラウザの IndexedDB にコピーされ保存されます。あとから PLAY で前回セッションを復元するとき参照されるのはこのコピーであり、ドロップ元のファイルパスは保持せず、ディスク上の元ファイルを自動で読み直すこともありません。<span class="app-manual-r255">DAW 等で波形・音声を更新したあとに本アプリへ最新版を反映したい場合は、必ずファイルをドロップし直してください。</span>ブラウザのストレージ整理や設定により IndexedDB が消えると、前回のトラックは復元できません。
- データ量の多いトラックを複数同時に扱うと、ブラウザのメモリ使用量が増え、操作や描画の応答が遅くなることがあります。長時間の連続再生のあとも同様です。タブをバックグラウンドに置いたり、ブラウザや OS の省電力・省リソースの働きでタブの優先度が下がったりすると、音声の再生そのものは続く場合でも、タイマーや `requestAnimationFrame` の更新頻度が下がり、メーターやシークバーなど画面表示の追従が鈍くなることがあります（ブラウザ仕様によるものです）。表示を確実に追いたいときはタブを前面に戻すか、一度ページを閉じて開き直してください。

## ソースコードを改変する場合

### ソースの読み方とユーザー初期値

- `js/main.js` 先頭付近の定数（ `DEFAULT_SPECTRUM_FLOOR_DB`・`DEFAULT_METER_FLOOR_DB`・`DEFAULT_FADE_TIME_SEC` ）で、スペクトラム／メーター表示の初期下限 dB とフェード時間（秒）の初期値を変更できます。dB は `DISPLAY_ANALYSIS_FLOOR_DB` に含まれる値に合わせてください。含まれない値にした場合は、スペクトラム・メーターとも -50 dB にフォールバックします。フェード秒数は 1〜30 の整数のみ（それ以外・空欄は `DEFAULT_FADE_TIME_SEC` を 1〜30 に収めた値に置き換え）。下限 dB の選択肢を増やすときは、`DISPLAY_ANALYSIS_FLOOR_DB` の配列と、トランスポート付近のセレクトの `<option>` を同じ集合になるよう揃えて更新してください。画面上で変更した表示下限・フェード秒数の保存先、および音声セッションの保存先については、取扱説明の「特記事項」「使い方」を参照してください。

### スペクトラム表示の実装

- 各列の中心周波数は、グラフ下の三方オクタ系に近い周波数ラベル（上下段を合わせた位置）と同じ並びに揃えています（ナイキスト周波数以下に入る分だけがバーとして表示され、多いときは 31 本になります）。帯域レベルは、マスター直後の Analyser の FFT から各帯の境界で電力を重み付け積分したあと、ごく弱い列方向ガウス（SPEC_BLUR_SIGMA）で隣列にわずかに漏らします（σ=0 だと単音が 1 列だけに閉じることがある）。SPEC_SKIRT_* はぼかし成分の距離別ブースト／減衰で、山の肩の形に強く効きます。単音らしいときはぼかし後の最大を時域ピーク dB に合わせる補正をかけます。
- バー本体には、計測用バルメータでいう準ピークに近い体感になるよう、上昇と落下で時定数の違う指数追随（SPEC_SPECT_QP_RISE_SEC・SPEC_SPECT_QP_FALL_SEC）をかけています。列ピーク表示の中心ホールド秒数は SPEC_SPECT_PEAK_HOLD_CENTER_SEC（L/R メーターの PEAK_HOLD_SEC とは別）、ホールド後の減衰は SPEC_PEAK_RELEASE_DB_PER_SEC 基準です。
- FFT と時域ピークのキャリブレーションで周波数ビンをわずかに補正し、例えば -20 dBFS の正弦波がスペクトラム上でおおよそ -20 dB に触れるようにしています（厳密ではありません）。ヘッドルームの固定クリップはかけず、実信号が 0 dBFS に近いときは 0 dB まで届き得ます。一方で、据え置き型アナライザで好まれる「ピークがすぐ満タンにならない」見え方として、SPEC_DISPLAY_PEAK_SOFT_KNEE_DB より上の帯だけ緩やかな非線形（SPEC_DISPLAY_PEAK_SOFT_GAMMA）で中間をやや低く見せています。
